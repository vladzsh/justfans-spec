# realtime

## ADDED Requirements

### Requirement: Обнаружение тихого обрыва соединения
WS-клиент SPA ДОЛЖЕН обнаруживать «тихий» обрыв (разрыв сети без события `close`) по отсутствию ответа `pong`. На каждый отправленный `ping` клиент ДОЛЖЕН ожидать `pong`; если `pong` не получен в пределах окна, кратного `heartbeat_seconds` (порог ≈ `2 × heartbeat_seconds`), клиент ДОЛЖЕН принудительно закрыть сокет и инициировать reconnect (с уже определёнными backoff+jitter и resync). Индикатор соединения ДОЛЖЕН отражать переход в состояние переподключения.

#### Scenario: Тихий обрыв сети
- **WHEN** сеть у чатера пропала без штатного закрытия сокета (нет события `close`) и `pong` перестал приходить
- **THEN** в пределах ≈`2 × heartbeat_seconds` индикатор уходит из «онлайн», клиент принудительно переподключается, после восстановления сети происходит resync без дублей

#### Scenario: Живое соединение не закрывается ложно
- **WHEN** соединение активно и `pong` приходит на каждый `ping`
- **THEN** клиент не закрывает сокет и остаётся в состоянии «онлайн»
