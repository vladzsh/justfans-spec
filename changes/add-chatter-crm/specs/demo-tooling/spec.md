# demo-tooling

## ADDED Requirements

### Requirement: Запуск одной командой
`docker compose up` в корне `justfans-backend` ДОЛЖЕН поднимать backend, PostgreSQL, Redis и frontend (nginx). Frontend собирается из git URL репозитория `justfans-frontend` (без ручного клонирования второго репо). После старта приложение доступно на одном порту (nginx), миграции применяются автоматически.

#### Scenario: Холодный старт
- **WHEN** жюри клонирует `justfans-backend` и выполняет `docker compose up`
- **THEN** через одну команду приложение открывается в браузере на документированном порту, вход работает

### Requirement: Сидер тестовых данных
Команда `manage.py seed` ДОЛЖНА создавать: 2–3 контент-модели, 3–4 чатеров, 1 тимлида, несколько диалогов на чатера с историей сообщений (включая PPV и диалоги с активным `awaiting_reply_since`). Учётные данные ДОЛЖНЫ быть фиксированными и задокументированными в README. Повторный запуск сидера НЕ ДОЛЖЕН дублировать данные.

#### Scenario: Прогон сидера
- **WHEN** выполнен `manage.py seed`
- **THEN** можно войти как любой чатер или тимлид по учёткам из README; у чатеров есть диалоги с историей, счётчиками непрочитанных и ожидающими фанами

#### Scenario: Повторный прогон
- **WHEN** сидер запущен второй раз
- **THEN** дубликатов пользователей/диалогов нет

### Requirement: Эмуляция входящего сообщения фана
`POST /api/demo/fan-message/` ДОЛЖЕН создавать входящее сообщение фана: с `conversation_id` — в указанный диалог, без — в случайный; `text` опционален (случайная фраза). В UI чатера ДОЛЖНА быть кнопка «симулировать сообщение фана» для открытого диалога — чтобы проверить realtime в одиночку.

#### Scenario: Проверка realtime в одиночку
- **WHEN** проверяющий нажимает кнопку симуляции (или дёргает эндпоинт через curl)
- **THEN** сообщение фана мгновенно появляется в переписке и списке диалогов чатера, а на мониторе тимлида появляется ожидающий диалог

### Requirement: README с инструкцией проверки
README `justfans-backend` ДОЛЖЕН содержать: команду запуска и порты, тестовые учётки (чатер и тимлид), способ эмуляции фана, описание env-порогов, ссылку на задеплоенное решение и ссылку на репозиторий спецификации. README `justfans-frontend` — запуск dev-сервера и ссылку на backend-репо.

#### Scenario: Жюри проверяет по README
- **WHEN** проверяющий следует разделу «Запуск» README
- **THEN** все шаги выполняются без обращения к автору: запуск, вход за чатера и тимлида, эмуляция фана

### Requirement: Деплой на Railway
Решение ДОЛЖНО быть задеплоено на Railway: backend (с прогнанными миграциями и сидером), PostgreSQL, Redis, frontend-nginx как публичный сервис, проксирующий `/api` и `/ws` на backend по приватной сети. Публичная ссылка и учётки — в README.

#### Scenario: Проверка деплоя
- **WHEN** проверяющий открывает ссылку из README
- **THEN** вход работает за чатера и тимлида, realtime-сценарии (сообщение, монитор, просрочка) воспроизводятся на живом стенде
