# Proposal: add-monitor-models-panel

## Why

Сейчас монитор тимлида отображает только срезы по чатерам. Модели без просроченных диалогов на мониторе невидимы — тимлид не видит их общую загрузку. Нужна агрегация по контент-моделям: сколько диалогов в работе у каждой модели и кто из фанов ждёт ответа.

## What Changes

- `GET /api/monitor/snapshot/` получает ключ `models` рядом с существующим `chatters`. Каждая запись — агрегация по всем диалогам модели независимо от того, какой чатер ведёт диалог.
- Событие `monitor.update` меняет форму payload: вместо плоского объекта чатера отдаётся `{"chatter": {...}, "models": [...]}`. Это breaking change контракта — обе стороны (backend и frontend) наши.
- Добавляется переиспользуемая функция `build_models_snapshot()` в `chat/services.py`; REST-снапшот и WS-broadcast используют её совместно, гарантируя консистентность.

## Capabilities

### Modified Capabilities

- `teamlead-monitor`: добавлена агрегация по моделям в снапшот и в `monitor.update`; изменена форма payload `monitor.update`.
