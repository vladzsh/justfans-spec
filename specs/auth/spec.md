# auth

## Requirements

### Requirement: Сессионный вход по логину и паролю
Система ДОЛЖНА аутентифицировать пользователя по `username`/`password` через `POST /api/auth/login/` и устанавливать Django-сессию (cookie). Ответ ДОЛЖЕН содержать `{id, username, display_name, role}`.

#### Scenario: Успешный вход
- **WHEN** отправлены корректные `username` и `password`
- **THEN** ответ 200 с данными пользователя, последующие запросы авторизованы сессионной cookie

#### Scenario: Неверные учётные данные
- **WHEN** отправлен неверный пароль
- **THEN** ответ 401 с `{"detail": ...}`, сессия не создаётся

### Requirement: Доступ по ролям
Каждый пользователь ДОЛЖЕН иметь роль `chatter` или `teamlead`. Эндпоинты чатера (`/api/conversations/*`, `/api/sync/`) ДОЛЖНЫ быть доступны только роли `chatter`; эндпоинты монитора (`/api/monitor/*`) — только роли `teamlead`. Неавторизованные запросы к любому `/api/*` (кроме login) ДОЛЖНЫ получать 401.

#### Scenario: Тимлид обращается к эндпоинту чатера
- **WHEN** пользователь с ролью `teamlead` запрашивает `GET /api/conversations/`
- **THEN** ответ 403

#### Scenario: Неавторизованный запрос
- **WHEN** запрос без сессии к `GET /api/conversations/`
- **THEN** ответ 401

#### Scenario: Маршрутизация фронтенда по роли
- **WHEN** пользователь вошёл в SPA
- **THEN** роль `chatter` попадает на рабочее место чатера, роль `teamlead` — на монитор; чужой маршрут недоступен (redirect)

### Requirement: Авторизация WebSocket той же сессией
WS-соединение `/ws/` ДОЛЖНО авторизоваться той же сессионной cookie (`AuthMiddlewareStack`). Соединение без валидной сессии ДОЛЖНО отклоняться с close code 4401.

#### Scenario: Подключение без сессии
- **WHEN** клиент открывает `/ws/` без валидной сессионной cookie
- **THEN** сервер закрывает соединение с кодом 4401

### Requirement: Текущий пользователь и выход
`GET /api/auth/me/` ДОЛЖЕН возвращать данные текущего пользователя (payload как у login). `POST /api/auth/logout/` ДОЛЖЕН завершать сессию.

#### Scenario: Восстановление сессии при загрузке SPA
- **WHEN** SPA загружается при живой сессии
- **THEN** `GET /api/auth/me/` возвращает 200 и пользователь попадает на свой интерфейс без повторного логина
